## DOCTYPE 的作用是什么？

### 问题

DOCTYPE 是什么？它在 HTML 中起什么作用？如果不写会有什么影响？

### 关键要点

* 告诉浏览器 HTML 使用的规范版本
* 决定标准模式还是怪异模式
* 影响 HTML / CSS / JS 的解析
* 必须写在文档第一行
* HTML5 写法固定

### 完整答案

DOCTYPE（Document Type Declaration，文档类型声明）用于告诉浏览器当前 HTML 文档采用的规范版本，使浏览器能够按照正确的标准解析和渲染页面。
DOCTYPE 的核心作用是**决定浏览器采用标准模式（CSS1 Compat）还是怪异模式（BackCompat）**。如果 DOCTYPE 缺失或声明错误，浏览器会进入怪异模式，导致布局、样式和脚本行为在不同浏览器中表现不一致。
在 HTML5 中，DOCTYPE 的标准写法为 `<!DOCTYPE html>`，并且**必须写在 HTML 文档的第一行**。

## 浏览器的标准模式和怪异模式有什么区别？

### 问题

什么是标准模式和怪异模式？它们是如何触发的？为什么必须使用标准模式？

### 关键要点

* 标准模式遵循 W3C 标准
* 怪异模式用于兼容旧网页
* 触发条件与 DOCTYPE 有关
* 怪异模式下行为不一致

### 完整答案

浏览器渲染页面主要有两种模式：标准模式（CSS1 Compat）和怪异模式（BackCompat）。
标准模式下，浏览器严格按照 W3C 标准解析 HTML 和 CSS，使用当前浏览器支持的最高标准，是现代 Web 开发推荐的模式。
怪异模式主要用于兼容早期未规范书写的旧网页，浏览器会以宽松、非标准的方式解析页面，不同浏览器的表现差异较大。
**当 DOCTYPE 不存在或声明不正确时，浏览器会进入怪异模式**。因此，为了保证页面在不同浏览器中的一致性，必须正确声明 DOCTYPE。

>W3C （World Wide Web Consortium，万维网联盟）
>制定 HTML、CSS 等 Web 技术规范的组织，浏览器在标准模式下会尽量按照 W3C 规范解析和渲染页面。

## 什么是 HTML 语义化？

### 问题

HTML 语义化是什么？为什么要强调语义化开发？

### 关键要点

* 用合适的标签表达内容含义
* 提高可读性和可维护性
* 有利于 SEO 和无障碍访问
* 有利于机器理解
::: tip 
SEO（Search Engine Optimization，搜索引擎优化）
SEO 是搜索引擎优化，通过合理的 HTML 结构、语义化标签和页面信息，让搜索引擎更容易理解网页内容，从而提高自然搜索结果中的排名。
:::
### 完整答案
HTML 语义化是指根据内容的结构和含义，选择语义合适的 HTML 标签来组织页面结构，而不是单纯使用 div 和 span 进行布局。
语义化的好处包括：代码结构清晰、可读性和可维护性高；搜索引擎可以更好地理解页面内容，有利于 SEO；屏幕阅读器等辅助设备可以更准确地解析页面；在无 CSS 的情况下页面仍具备合理结构。

## section 和 article 的区别是什么？

### 问题

section 和 article 都是语义化标签，它们的使用场景有什么不同？

### 关键要点

* article 表示独立完整内容
* section 表示结构性区块
* article 具有可复用性

### 完整答案

`article` 表示一段**独立、完整、可脱离页面单独存在的内容**，例如文章、博客帖子、新闻、评论等。
`section` 表示文档中的一个结构性区块或章节，用于对内容进行逻辑分组，但本身不一定是独立内容。
简单来说，如果内容可以单独拿出去复用，优先使用 `article`，否则使用 `section`。

## meta 标签的作用和分类有哪些？

### 问题

meta 标签有什么作用？主要分为哪几类？

### 关键要点

* 提供页面元数据
* 不直接显示在页面
* name / http-equiv / content

### 完整答案

`meta` 标签用于描述 HTML 文档的元数据，不会直接显示在页面中。
其主要作用包括搜索引擎优化、字符集设置、页面刷新与跳转、缓存控制以及页面行为控制。
meta 标签主要分为三类：

* 使用 `name` 描述页面信息，如 keywords、description、author、robots
* 使用 `http-equiv` 模拟 HTTP 响应头，如 Expires、Refresh、Content-Type
* `content` 用于配合前两者提供具体内容

## meta charset 的作用是什么？

### 问题

`<meta charset="UTF-8">` 的作用是什么？为什么必须写？

### 关键要点

* 指定字符编码
* 防止乱码
* 影响浏览器解析文本

### 完整答案

`meta charset` 用于声明 HTML 文档所使用的字符编码格式。
浏览器在解析 HTML 文本时会根据该声明对字符进行解码，如果编码不一致就会产生乱码问题。
在现代 Web 开发中，通常统一使用 UTF-8 编码，因此需要在页面中显式声明。

## src 和 href 的区别是什么？

### 问题

src 和 href 有什么本质区别？为什么 src 会阻塞渲染？

### 关键要点

* src 是资源嵌入
* href 是资源引用
* src 会下载并执行
* href 不阻塞解析

### 完整答案

`src` 表示将外部资源嵌入到当前文档中，资源会替换当前元素本身，例如 script、img、iframe。浏览器在解析到 src 时会立即下载、解析并执行该资源，因此会阻塞页面渲染。
`href` 表示建立当前文档与外部资源之间的关系，例如 a、link。资源通常并行加载，不会阻塞 HTML 解析。
因此，src 用于必需资源的嵌入，href 用于资源的引用和关联。

## 行内元素和块级元素的区别是什么？

### 问题

行内元素和块级元素有哪些核心区别？

### 关键要点

* 是否独占一行
* 是否可设置宽高
* 子元素包含能力

### 完整答案

行内元素不会独占一行，多个元素可以在同一行显示，其宽高由内容决定，通常不能直接设置宽高，只能包含文本或行内元素。
块级元素会独占一行，可以设置宽高和内外边距，宽度默认撑满父容器，并且可以包含任意类型的元素。
通过 `display: inline-block` 可以将元素转换为既不独占一行又能设置宽高的行内块元素。

## 为什么推荐使用 link 而不是 @import？

### 问题

link 和 @import 引入 CSS 有什么区别？为什么开发中推荐 link？

### 关键要点

* 加载时机不同
* link 可被 JS 控制
* link 权重更高
* 兼容性更好

### 完整答案

`link` 是 HTML 标签，在页面加载时同步加载 CSS，不存在兼容性问题，并且可以通过 JavaScript 动态控制。
`@import` 是 CSS 语法，必须在 CSS 文件中使用，页面加载完成后才会加载，加载顺序靠后，权重较低，且早期浏览器兼容性较差。
因此，在实际开发中推荐使用 `link` 引入样式文件。

## async 和 defer 的区别是什么？

### 问题

async 和 defer 有什么共同点和核心区别？各自适合什么场景？

### 关键要点

* 都是异步下载
* 都不阻塞 HTML 解析
* 执行时机不同
* 顺序是否确定

### 完整答案

async 和 defer 都可以在不阻塞 HTML 解析的前提下异步加载脚本。
async 脚本在下载完成后立即执行，执行顺序不确定，适合彼此独立、不依赖 DOM 的脚本。
defer 脚本在 DOM 解析完成后、`DOMContentLoaded` 事件之前按顺序执行，适合依赖 DOM 结构的脚本。
#### 解释
当`<script>` 标签加上 async 属性时，浏览器会在后台异步下载该脚本。
下载完成后立即执行，无视 HTML 中其他脚本的顺序，可能在 DOM 还未解析完成时就执行。
适合独立的第三方库或统计脚本，比如 `Google Analytics`。
```html
<script src="analytics.js" async></script>
```
但当`<script>` 标签加上 defer 属性时，浏览器同样会异步下载脚本，但是会等到 HTML 文档完全解析完成后再执行，并且按照在 HTML 中出现的顺序执行。
适合依赖 DOM 的脚本，例如页面功能初始化脚本。

## 浏览器渲染流程简述

### 问题

浏览器从获取 HTML 到页面展示，核心渲染流程是什么？

### 关键要点

* DOM
* CSSOM
* Render Tree
* Layout
* Paint

### 完整答案

浏览器首先解析 HTML 构建 DOM 树，然后解析 CSS 构建 CSSOM 树。
DOM 树和 CSSOM 树合并生成 Render Tree，接着进行布局计算（Layout），最后进行绘制（Paint），将页面呈现在屏幕上。
>CSSOM 树是浏览器解析 CSS 后生成的对象模型树，它记录了每个元素对应的样式信息。浏览器会将 CSSOM 与 DOM 树结合生成 Render Tree，从而计算布局和绘制页面。
