# 刷题笔记排版 Skill

## 触发条件

当用户要求整理、排版刷题笔记时使用。先阅读 [examples.md](examples.md) 获取完整的输入→输出示例。

## 第一步：识别文档结构

刷题笔记通常包含以下区块，按顺序排列：

| 区块 | 说明 |
|------|------|
| **页头** | 标题 + 来源 + 视频链接 + 题目表格 |
| **例题** | 视频中讲解的核心题目 |
| **作业题** | 课后练习题 |
| **附录**（可选） | 额外知识补充（如正则、API 用法等） |

## 第二步：通用规则

### 页头格式

保留原有结构，确保：
1. 一级标题（专题名称）
2. 来源引用行（`> 来源：...`）
3. 视频链接
4. 题目表格（保持原样，不改动）

### 每道题的模板

```markdown
### 题目名称

- 思路要点1
- 思路要点2
- 思路要点3

```js
// 代码
```

> 时间复杂度：O(n)，空间复杂度：O(1)
```

**模板说明：**
- 三级标题用题目名称（带 LeetCode 链接更佳，但不强制添加）
- 思路要点用无序列表，提炼核心思路，保留用户原始表述风格
- 代码块保留原样（包括 console.log 调试语句）
- 代码块下方补充时间/空间复杂度分析（如果原文没有，适当补充）

### 章节层级

- `## 例题` 和 `## 作业题` 作为二级标题分隔
- 每道题用 `###` 三级标题
- 如果有附录知识补充，用 `## 附录` 作为二级标题

### 思路要点整理规则

1. 从用户的注释和描述中提炼核心思路，形成简洁的列表
2. 保留用户记录的踩坑点和注意事项（如"不知道怎么处理相等的情况"→ 保留为踩坑记录）
3. 如果用户记录了多种解法，每种解法单独一个三级标题（如 `### 接雨水 - 前后缀分解`、`### 接雨水 - 双指针`）
4. 适当补充：为什么这样做（一句话原理），但不要大段解释

### 代码处理

- **保留 console.log**：不删除调试语句
- **保留原始注释**：用户写的注释原样保留
- **确保语言标注**：代码块必须标注 `js`/`python` 等
- **去除多余空行**：代码块内连续空行压缩为一个

### 附录处理

额外知识补充（如正则表达式、API 用法等）保留在原文末尾，作为 `## 附录` 章节。内容不做大幅删改，仅统一格式。

## 第三步：增强内容（适度）

在不改变用户原始内容的前提下，适当增强：

1. **复杂度分析**：每道题代码下方补充时间/空间复杂度（如果原文缺失）
2. **思路总结**：将散落在注释中的思路提炼为结构化列表
3. **踩坑标记**：用户提到的易错点，用加粗或特殊标记突出

## 第四步：验证清单

排版完成后检查：

- [ ] 页头完整（标题 + 来源 + 视频 + 题目表格）
- [ ] 例题和作业题用二级标题分隔
- [ ] 每道题有三级标题 + 思路列表 + 代码
- [ ] 代码块都有语言标注
- [ ] 多余空行已清理（代码块内外）
- [ ] 附录知识独立成章节
- [ ] 复杂度分析已补充
- [ ] 用户原始内容未被删减
